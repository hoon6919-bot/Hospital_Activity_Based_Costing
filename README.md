# ClinicProfit AI (의료기관 원가계산 시스템)

병원 및 의료기관의 복잡한 원가를 활동기준원가계산(ABC, Activity-Based Costing) 방식을 통해 부서별/진료과별/의사별로 정확하게 배부하고 분석하여, 숨은 원가를 찾아내고 수익성을 개선하는 **로컬 풀스택(Local Full-Stack) 웹 애플리케이션**입니다.

---

## 🚀 주요 기능 및 시스템 흐름

본 시스템은 단순한 데이터 저장을 넘어, 병원의 재무 데이터를 입력받아 고도화된 4단계 배부 로직을 거쳐 최종 손익 리포트를 생성하는 일련의 과정을 자동화합니다.

1.  **데이터 수집:** 기초코드, 사원소속, 수익, 비용, 환자통계 등 원천 데이터 입력
2.  **규칙 설정:** 배부 기준(드라이버) 및 원가 배부 로직(계정 ➔ 활동 ➔ 타부서 ➔ 최종 진료과) 매핑
3.  **엔진 구동:** 사용자가 설정한 로직을 바탕으로 ABC 계산 엔진 실행
4.  **결과 시각화:** 요약 차트 및 상세 결과표를 통한 수익성 분석

## 🖥 화면별 상세 기능 명세 (Screen Details)

팀원 및 이해관계자가 시스템의 흐름을 쉽게 파악할 수 있도록 각 메뉴별 주요 기능과 역할을 정리했습니다.

### 1. 홈 (`/`) - Dashboard
*   **역할:** 시스템 접근 시 가장 먼저 만나는 대시보드 화면입니다.
*   **기능:** 시스템의 주요 가치(AI 기반 숨은 원가 발굴) 제안과 함께, 전체 원가계산 절차(데이터 입력 ➔ 로직 설정 ➔ 실행 ➔ 결과 분석)를 한눈에 볼 수 있는 가이드 및 내비게이션을 제공합니다.

### 2. 기초코드 (`/setup`) - Master Data
*   **역할:** 원가계산의 뼈대가 되는 마스터 데이터를 정의합니다.
*   **기능:** 병원 내 모든 부서, 직종, 계정코드, 표준 활동 등을 정의하고 관리합니다. 모든 배부 로직은 이곳에 정의된 코드를 기준으로 작동합니다.

### 3. 사원소속 (`/employee`) - Employee Data
*   **역할:** 병원 구성원들의 기본 정보와 인건비를 관리합니다.
*   **기능:** 사원별 소속 부서, 직종, 직급, 그리고 각 사원에게 발생한 인건비(급여) 데이터를 입력하고 저장합니다. 이 데이터는 인건비 배부의 기초 자료가 됩니다.

### 4. 수익 (`/revenue`) - Revenue Data
*   **역할:** 진료과별, 의사별 수익 데이터를 집계합니다.
*   **기능:** 환자 진료 행위로 인해 발생한 수익 데이터를 입력(수작업 또는 엑셀 업로드)하며, [데이터 요약] 탭을 통해 입력된 수익의 전체 규모를 확인합니다.

### 5. 비용 (`/cost`) - Cost Data
*   **역할:** 병원 운영에 발생한 모든 비용 데이터를 부서별/계정별로 집계합니다.
*   **기능:** 인건비, 재료비, 관리운영비 등의 원천 비용 데이터를 입력합니다. 이 비용들이 향후 배부 로직을 타고 최종 진료과로 분배됩니다.

### 6. 환자통계 (`/patient-stats`) - Patient Statistics
*   **역할:** 진료과별 입원/외래 환자 수, 재원일수 등의 통계 지표를 관리합니다.
*   **기능:** 환자 통계 데이터는 진료지원부서(예: 원무과)의 비용을 각 진료과로 배부할 때 가장 핵심적인 '배부 기준(드라이버)'으로 사용됩니다.

### 7. 원가배부값 (`/allocation-value`) - Allocation Drivers
*   **역할:** 원가를 나누는 기준이 되는 '드라이버(Driver)'의 실제 수치를 정의합니다.
*   **기능:** 인원수, 면적, 처방건수 등 비용을 쪼개기 위한 기준 지표들을 관리합니다. 각 부서별로 드라이버 값이 어떻게 분포되어 있는지 요약 및 결과표를 제공합니다.

### 8. 계산로직 (`/logic`) - ABC Logic Engine Rules
*   **역할:** 시스템의 가장 핵심적인 비즈니스 규칙(배부 룰)을 설정합니다.
*   **기능:**
    *   **계정 매핑:** 특정 비용 계정을 어느 부서로 어떻게 보낼지 정의
    *   **활동 비율:** 부서 내 인력이나 비용이 어떤 '표준 활동'에 몇 %씩 투입되는지 비율 정의
    *   **활동 로직:** 집계된 활동 원가를 최종 원가대상(진료과 등)으로 배분하기 위한 최종 규칙 설정

### 9. 실행 (`/run`) - Execution Engine
*   **역할:** 설정된 데이터와 로직을 바탕으로 실제 원가계산을 구동하는 컨트롤 타워입니다.
*   **기능:** 대상 연월을 선택하고 [원가계산 시작] 버튼을 누르면, 프론트엔드에 내장된 4단계 알고리즘 엔진이 작동하여 모든 배부 계산을 순차적으로 수행합니다.

### 10. 결과 (`/result`) - Result & Analytics
*   **역할:** 원가계산이 완료된 최종 손익 데이터를 시각화하여 분석합니다.
*   **기능:**
    *   **결과 요약:** 진료과별 수익, 원가, 이익을 한눈에 비교할 수 있는 막대형/파이형 차트 제공
    *   **상세 결과표:** 각 부서/의사별로 발생한 직접비, 간접비 배부액, 최종 손익을 상세한 표 형태로 제공
    *   **원가 배부 결과:** 어떤 부서의 비용이 누구에게 얼마만큼 흘러갔는지 추적(Tracking) 가능한 내역 제공

## 🗄️ 저장소 매핑 (DB Table & Store Key)

각 화면에서 다루는 데이터가 물리적으로 `database.sqlite` 내 어느 테이블과 Key에 저장되는지 매핑한 정보입니다.

| 메뉴 (화면) | 경로(URL) | DB 테이블 | 저장소 Key (`generic_store`) |
| :--- | :--- | :--- | :--- |
| **기초코드** | `/setup` | `generic_store` | `clinic_departments` 등 |
| **사원소속** | `/employee` | `generic_store` | `clinic_employees` |
| **수익** | `/revenue` | `generic_store` | `clinic_revenue_data` |
| **비용** | `/cost` | `generic_store` | `clinic_cost_data` |
| **환자통계** | `/patient-stats` | `generic_store` | `clinic_patient_stats` |
| **원가배부값** | `/allocation-value` | **`driver_data` (전용)** | - |
| **계산로직** | `/logic` | **`allocation_rules` (전용)** | - |

*(※ `generic_store`는 다목적 만능 금고 테이블로, 입력된 데이터가 JSON 형태로 묶여 한 줄로 통째로 저장되는 구조입니다.)*

## 🛠 기술 스택 (Tech Stack)

*   **Frontend:** React, Vite, CSS Modules, Lucide React (아이콘), Recharts (차트 시각화)
*   **Backend:** Node.js, Express (API 서버)
*   **Database:** SQLite (`database.sqlite` 기반 로컬 영구 저장소)

## 💻 실행 방법 (How to run)

현재 이 프로젝트는 프론트엔드와 백엔드를 동시에 실행하는 로컬 풀스택 개발 환경입니다.

```bash
# 1. 의존성 패키지 설치
npm install

# 2. 프론트엔드 및 백엔드 서버 동시 실행
npm run dev
```

*   **웹 애플리케이션 접속:** `http://localhost:5173`
*   **백엔드 API 서버 (데이터 저장처리):** `http://localhost:3001`
